ANIME.JS

An animation plugin for Javascript

ABOUT ANIME.JS

Anime.js is an animation plugin for Javascript, and is important to get to know as it can be a powerful tool to make your website more professional and attractive to users. Also, getting familiar with this plugin may provide a helpful learning experience in learning how to quickly adapt to plugins!

OBJECTIVES

To familiarize with Anime.js's objects by building a manga chapter reader. This will include navigating between pages, and displaying a previous page, current page, and next page. This tutorial will also include a loading animation, allow previews of the previous and next page when hovered, and the animation of the current page onto the main part of the screen.

REQUIREMENTS

  • Notepadd++ | Visual Studio | (or any general text editor)
  • Basic knowledge of CSS, HTML, Javascript (DOM)

TUTORIAL NOTE

This tutorial will NOT be going over much (if any) of the CSS portion of the code, as this tutorial is geared towards the familiarization of Anime.js. To learn more about CSS, please visit the provided link in the requirements section.

STEP FOUR: Animate load

We will first animation the loading of the page using the input element we placed in the introduction section! To do this, we will first need to review how the animation syntax of Anime.js looks like! To animate, you need to code "anime({ ... });" first! (In other words, you will be creating anime functional objects) Next we will look at a few properties that can be manipulated to get the desired animation!
  • TARGETS - elements, object, etc. to be selected for animation
    • '#loading'
  • VALUE - value of element(s) to be animated to as it changes
    • '100%'
  • EASING - an animation parameter, specifying the easing type of the animation
    • 'easeInOutExpo'
  • DURATION - how long the animation will take (in milliseconds)
    • 3000
  • ROUND - rounding the value (remove decimals)
    • 1
NOTE: How each property name ends with a colon, and the assign value ends with a parentheses! What it should look like now

STEP FIVE: Let's start reading

Next we will look at animation options.
  • UPDATE - will run the functional object every frame of the animation
  • COMPLETE - will run once the animation is complete (will not work for infinite loops)
  • BEGIN - will run once the animation has started
For this part of the demo, we will want to replace the loading input element with a button to start reading the manga chapter itself! Therefore, after the animation has ended, we will want to get the input element, remove it, and create a button with the id "start", and insert it where the animated input element was.
Afterward, we will attach a click event listener to the start button, which once clicked, will make all the other buttons/sections of the page that were previously hidden visible again, and delete the introduction section (along with the start button itself). What it should look like now The previous and next page divs on the left have had their opacity lowered. All three divs have an inital background of solid white. But this will change as each will be assigned a background image!

STEP SIX: Animate current page

We will now animate the current page div.

NOTE: Each page div is set up so that the ratio of the div size is the same as the original page and therefore can be easily scaled up in the animation object. This may require some simple math calculations. If dealing with different sized pages, this CSS approach may need some revision, but the page images in this demo are all the same dimension. In addition, the page divs have been translated down the webpage's side for a more professional look, though this is not essential.

To animate the current page div, we will be revisiting the animation object we wrote in the previous step. Similar to this step, we will be animating the current page div so that it is translated horizontally and vertically to the center of the page, while also scaling it up for readability. Some new animation parameters introduced in this step are:
  • SCALE - scaling the selector up or down
  • AUTOPLAY - specifies whether or not the animation automatically runs
  • TRANSLATEX - translates how far right or left the selector will be moved
  • TRANSLATEY - translates how far up or down the selector will be moved
We will also be attaching this animation as the event when the next page button is clicked. This code will be placed in the complete parameter option of the previous animation object we wrote: Notice the ".restart" method in the assignment of a click event listener. This means that the displayPage animation will play every time the button is clicked. Here are some more options offered by Anime.js!
  • RESTART - restart and play the animation
  • PLAY - plays the animation once, cannot be played again using this button / event listener
  • PAUSE - pauses the animation

STEP SEVEN: Navigation Buttons

To display the next page of the manga chapter, we will need to do 3 steps. First, we will need a string array of all the pages of the manga chapter. In the demo, I made an array contain all pages of Noragami's chapter 57 (approximately 44 pages). Next, we will need to create a (normal Javascript) function to update the divs to display the pages we want. To do this, I have created a variable (pageNum) to hold the current place of the current page in the manga page array. In the anime object (in which I named displayNewPage), we will update each div with the appropriate element in the manga page array. NOTE: The previous page update to its background image is checked in the case that when the user clicks the previous page navigation button, that the current page is now the very first page. In this case, the background image for the previous page div is removed, and the div returns to its solid white background again (as it was when first starting the manga reader)

We are not done this step just yet! This function must be called every time the user clicks navigation button, but also when the user first clicks the start button! Therefore, we must increment the pageNum variable (thus the reason why it was initiated as -1) and run the displayNewPage function when the start button is clicked! What it should look like now

STEP EIGHT: Preview Next and Previous Pages

We will now focus on the navigation buttons. Each button will have two click events attached to it, one for running the displayPage animation object, and the other to update pageNum.

Let's start with the next page button!

First attach a click event handler to the next button (using it's ID) with the displayPage animation object. We will also attach a second click even handler to update the pageNum variable. For the next page button, we must take care of the case in which there is no next page in the manga chapter. In this case, pageNum+1 would equal null. If it is null, send a alert message to the user indicating the end of the chapter, and do not run the displayPage animation object. If this isn't the case, simply increment pageNum, and run displayPage to update the page divs. The code for the previous page button will be very similar. However, instead of looking for the end of the chapter, we will look for the case in which there is not previous page for the current displayed page. In this case, pageNum would equal 0 (first element of manga page array). Code an alert message for the user that the current displayed page is the first page of the chapter, and do nothing else. If pageNum is not zero (greater than), decrement pageNum, and run displayPage to update the page divs. What it should look like now

STEP NINE:

Almost done! Let's add some more fancy animations to our previous page and next page divs! For this animation, we are going to make it so that when the user clicks on one of these divs, that div will scale up (but not as large as the current page div), and will need to be translated to the right and from the top/bottom to account for that scaling. In addition, the div's opacity will change as it is animated. It's opacity will increase as it is scaled for preview, and decrease as it reverts back to its original size and position. To do this, we must first introduce another animation parameters:
  • DIRECTION - specifies the direction of the animation (either 'alternate' or 'reverse')
In this animation, we will be using the alternate value for the direction property! We will also attach a click event listener to the next page button, increase opacity at the begining of the animation, and decrease the opacity at the completion of the animation. The code for the animation of the previous page div is the same, except the translationY property will likely need to be negative so that the page will be in view at the current position in the window, otherwise the user would have to scroll down the page will may get messy! Again, be sure to attach a click event listener! What it should look like now
